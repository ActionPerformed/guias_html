¿Qué es el Diseño Responsive?

El diseño responsive permite que una página web se adapte automáticamente al tamaño de la pantalla del dispositivo, sin necesidad de hacer zoom ni scroll horizontal.

❌ Sin responsive
🖥️
OK – 100%
📱
~30% visible
📟
~60% visible
✅ Con responsive
🖥️
OK – 3 columnas
📱
OK – 1 columna
📟
OK – 2 columnas
Estructura del proyecto: La actividad compara dos carpetas: No Responsive/ (CSS propio) y Responsive/ (Bootstrap). Analiza las diferencias entre ambas versiones.

2.4 Componentes Bootstrap

Navbar, Cards — listos para usar

Navbar — Barra de navegación con menú hamburguesa automático:

html
<nav class="navbar navbar-expand-lg navbar-light bg-light">
  <a class="navbar-brand" href="#">Logo</a>
  <button class="navbar-toggler" data-toggle="collapse"
          data-target="#menu">
    <span class="navbar-toggler-icon"></span>
  </button>
  <div class="collapse navbar-collapse" id="menu">
    <ul class="navbar-nav">
      <li class="nav-item"><a class="nav-link" href="#">Inicio</a></li>
    </ul>
  </div>
</nav>

Card — Estructura de tarjeta con cabecera, cuerpo y pie:

.card-img-top

.card-subtitle

.card-footer → Más información

03

Análisis Comparativo

Examina cada cambio concreto entre la versión fija y la versión responsive.

3.1 — El <head>: Añadir Bootstrap
❌ Antes — Sin Bootstrap
✅ Ahora — Con Bootstrap
❌ No Responsive
html
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Mi Página</title>
  <!-- SIN Bootstrap → CSS propio solamente -->
</head>
Problema: El navegador móvil escala la página entera al 30% para que "quepa". Todo se ve diminuto e ilegible.
✅ Responsive
html
<head>
  <meta charset="UTF-8">
  <!-- CRUCIAL: indica al móvil que use el ancho real -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Mi Página</title>

  <!-- 1. Bootstrap CSS -->
  <link rel="stylesheet" href="css/bootstrap.min.css">
  <!-- 2. Google Fonts -->
  <link href="https://fonts.googleapis.com/..." rel="stylesheet">
  <!-- 3. Font Awesome (iconos) -->
  <link rel="stylesheet" href="https://use.fontawesome.com/...">
  <!-- 4. Estilos propios al final -->
  <link rel="stylesheet" href="css/styles.css">
</head>
3.2 — Navbar: Menú Hamburguesa
❌ Antes — HTML simple
✅ Ahora — Navbar Bootstrap
❌ No Responsive
html
<header>
  <nav>
    <a href="#!"><img src="img/logo.png"></a>
    <div id="barra-navegacion">
      <ul>
        <li><a href="#!">INICIO</a></li>
        <li><a href="#!">LA LABORAL</a></li>
      </ul>
    </div>
  </nav>
</header>
✅ Responsive — con hamburguesa automática
html
<header class="container">
  <nav class="navbar navbar-expand-lg navbar-light bg-light">

    <!-- Logo -->
    <a class="navbar-brand" href="#!">
      <img src="img/logo.png" alt="Logo">
    </a>

    <!-- Botón hamburguesa (solo en móvil) -->
    <button class="navbar-toggler" type="button"
            data-toggle="collapse"
            data-target="#barra-navegacion"
            aria-expanded="false"
            aria-label="Botón de navegación">
      <span class="navbar-toggler-icon"></span>
    </button>

    <!-- Menú colapsable -->
    <div class="collapse navbar-collapse" id="barra-navegacion">
      <ul class="navbar-nav mr-auto mt-2 mt-md-0 text-center">
        <li class="nav-item active">
          <a class="nav-link" href="#!">INICIO</a>
        </li>
      </ul>
    </div>
  </nav>
</header>
Clase claveFunción
navbar-expand-lgMenú horizontal en lg+, hamburguesa en <lg
navbar-togglerBotón que aparece solo en móvil
collapse navbar-collapseMenú que se colapsa/expande
data-target="#id"Vincula el botón con el menú
3.3 — Sección Imágenes: Grid de Columnas
❌ Antes — Sin grid
✅ Ahora — Bootstrap Grid
❌ Sin control de layout
html
<div>
  <div>
    <div>Texto izquierda</div>
  </div>
  <div>Imagen derecha</div>
</div>
<!-- Sin clases = sin control responsive -->
✅ Layout adaptable automáticamente
html
<div class="container-fluid">
  <div class="row">
    <!-- Columna de texto: 100% en móvil, 50% en tablet+ -->
    <div class="col-md-6 bg-primary d-flex align-items-center">
      <div class="row justify-content-center">
        <div class="col-sm-8 text-center">
          <h2>Título</h2>
          <p>Descripción...</p>
        </div>
      </div>
    </div>
    <!-- Columna imagen: 100% en móvil, 50% en tablet+ -->
    <div class="col-md-6 bg-laboral1"></div>
  </div>
</div>
3.4 — Cards: Componente Bootstrap
❌ Antes — Div plano
✅ Ahora — Bootstrap Card
❌ Sin estructura semántica
html

  <img src="img/clase1.jpg" alt="...">
  <div>20 / Mayo / 2022</div>
  <div>
    <h3>Título</h3>
    <p>Descripción...</p>
  </div>
  <div><a href="#">Más Información</a></div>

✅ Card responsiva con altura uniforme
html

  <div class="card h-100">
    <img class="card-img-top" src="img/clase1.jpg" alt="...">
    <div class="card-header bg-primary">20 / Mayo / 2022</div>
    <div class="card-body">
      <h3 class="card-title">Título</h3>
      <h6 class="card-subtitle mb-2 text-muted">Subtítulo</h6>
      <p class="card-text">Descripción...</p>
    </div>
    <div class="card-footer">
      <a href="#" class="card-link">Más Información</a>
    </div>
  </div>
</div>
3.5 — Orden de Columnas: order-*

Bootstrap permite invertir el orden visual de las columnas sin tocar el HTML. Útil para poner la imagen encima en móvil aunque en el HTML vaya después del texto.

📱 Móvil
Columna A (Texto) HTML: 1º | order-1 / order-md-2
Columna B (Imagen) HTML: 2º | order-2 / order-md-1
html
<div class="row">
  <!-- En HTML va 1ª, pero en desktop aparece 2ª -->
  <div class="col-12 col-md-6 order-1 order-md-2 bg-secondary">Texto</div>

  <!-- En HTML va 2ª, pero en desktop aparece 1ª -->
  <div class="col-12 col-md-6 order-2 order-md-1 bg-laboral2">Imagen</div>
</div>
04

Flujo de Responsive

El diseño responsive sigue un proceso lógico de tres pasos que siempre se repite.

📐
1. Breakpoints
<576px → xs
≥576px → sm
≥768px → md
≥992px → lg
≥1200px → xl
📱
2. Mobile First
Diseña para móvil
Añade breakpoints
hacia pantallas
más grandes
⊞
3. Clases Responsive
col-12 → 100%
col-sm-6 → 50%@sm+
col-md-4 → 33%@md+
col-lg-3 → 25%@lg+
Mobile First: Sin prefijo (col-6) aplica a todos los tamaños. Con prefijo (col-md-6) solo aplica desde ese breakpoint hacia arriba. Por eso se empieza por móvil.

Añade un nuevo div class="col-md-4" después del tercer card

02
Cambiar el breakpoint del navbar
📄 index.html
1
El navbar actual usa navbar-expand-lg: hamburguesa aparece en pantallas < 992px
2
Modifícalo para que el hamburguesa aparezca también en tablets (≥768px)
💡 Cambia navbar-expand-lg por navbar-expand-md
html — Solución
<nav class="navbar navbar-expand-md navbar-light bg-light">
<!-- Cambiado: navbar-expand-lg → navbar-expand-md
     Ahora hamburguesa aparece en <768px
     En tablets ≥768px el menú ya es horizontal -->
03
Modificar el layout para 4 cards por fila en desktop
📄 index.html
1
Las cards de eventos usan actualmente col-md-6 col-lg-4 → 3 por fila en desktop
2
Modifica para que en desktop aparezcan 4 cards por fila
💡 12 columnas ÷ 4 cards = 3 columnas cada una → col-lg-3
html — Solución
<div class="col-md-6 col-lg-3 pb-2">
<!-- Clave: col-lg-3
     12 ÷ 3 = 4 columnas por fila en desktop
     En tablet (md) siguen siendo 2 por fila (col-md-6) -->
04
Centrar el footer con Flexbox
📄 index.html y/o css/styles.css
1
Localiza el row dentro del <footer>
2
Añade las clases de alineación Flexbox al row
💡 Añade align-items-center justify-content-center al row del footer
html — Solución
<div class="row align-items-center justify-content-center">
<!-- align-items-center = centrado vertical
     justify-content-center = centrado horizontal -->
05
Cambiar el color de la cabecera de las cards
📄 css/styles.css
1
Abre css/styles.css
2
Añade una regla para .card-header y cambia el color de fondo
css — Solución
.card-header {
  background-color: #28a745 !important;
  /* Verde. !important sobreescribe bg-primary de Bootstrap */
}
06
Invertir los colores del banner sin tocar el HTML
📄 css/styles.css
1
En el banner principal hay dos secciones con fondos: .imagen-superior y .imagen-inferior
2
Intercambia sus colores/fondos solo en CSS, sin modificar el HTML
💡 Busca .bg-laboral1 y .bg-laboral2 o las clases de imagen en styles.css
css — Solución
.imagen-superior {
  background-color: #FA5858; /* Era #2E9AFE (azul) → ahora rojo */
}

.imagen-inferior {
  background-color: #2E9AFE; /* Era #FA5858 (rojo) → ahora azul */
}
