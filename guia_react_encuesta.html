Conceptos Fundamentales

🧩

3.1 Componentes

Funciones que devuelven JSX

Un componente es una función JavaScript que retorna JSX — una sintaxis que parece HTML pero es JavaScript.

Reglas: siempre empiezan con Mayúscula, retornan JSX, y pueden recibir props.

Componente padre (App):

App.jsx
function App() {
  // Lógica del componente
  return <div>...</div>
}

Componente hijo (Option):

Option.jsx
function Option({ texto, votos, onVotar }) {
  return (
    <div className="option-card">
      <span>{texto}</span>
      <span>{votos}</span>
      <button onClick={onVotar}>Votar</button>
    </div>
  )
}
📦

3.2 Props (Propiedades)

Argumentos que el padre pasa al hijo

Las props son la única forma de que el padre envíe datos al hijo. El hijo los recibe como parámetros y no puede modificarlos.

⬇ Padre — envía props
⬆ Hijo — recibe props
App.jsx — el padre
jsx
// App.jsx — el PADRE pasa props al hijo
<Option
  texto="Opción A"
  votos={5}
  onVotar={() => handleVoto(1)}
/>
Option.jsx — el hijo
jsx
// Option.jsx — el HIJO recibe las props
function Option({ texto, votos, onVotar }) {
  // texto   = "Opción A"
  // votos   = 5
  // onVotar = función del padre
}
Analogía: Las props son como los argumentos de una función, pero para componentes visuales.
⚡

3.3 Estado con useState

Datos que cambian y provocan re-renders

El estado son datos que pueden cambiar durante la vida del componente. Cuando cambian, React re-renderiza el componente automáticamente.

jsx
import { useState } from 'react'

function App() {
  // useState devuelve: [valorActual, funciónParaActualizar]
  const [opciones, setOpciones] = useState([
    { id: 1, texto: 'Opción A', votos: 0 }
  ])
  return <div>...</div>
}
❌ Incorrecto
✅ Correcto
Mutación directa — NUNCA hacer esto
jsx
// ❌ Modifica el estado directamente → React no se entera
opciones.push(nuevoElemento)
opciones[0].votos++
Actualización inmutable — React detecta el cambio
jsx
// ✅ Crea nuevo array — React detecta el cambio y re-renderiza
setOpciones(opciones.map(op => ...))
🔁

3.4 Renderizado de Listas con map()

Transforma un array en elementos JSX

Para renderizar listas dinámicas, usamos map(). Cada elemento necesita una key única para que React optimice los re-renders.

jsx
{opciones.map(opcion => (
  <Option
    key={opcion.id}         // ← obligatorio
    texto={opcion.texto}
    votos={opcion.votos}
    onVotar={() => handleVoto(opcion.id)}
  />
))}
ParteFunción
opciones.map(...)Itera cada elemento del array
opcion => (...)Transforma cada elemento en JSX
key={opcion.id}Identificador único — React lo necesita para reconciliación
<Option .../>JSX que se renderiza por cada opción
Sin key, React muestra un aviso en consola y puede comportarse de forma inesperada al actualizar la lista.
📡

3.5 Funciones Callback como Props

Comunicación del hijo al padre

Las props solo bajan (padre → hijo). Para que el hijo "informe" al padre de algo, el padre le pasa una función callback como prop. El hijo la llama cuando ocurre el evento.

App.jsx — define y pasa la función:

jsx
const handleVoto = (id) => {
  setOpciones(opciones.map(...))
}

<Option
  onVotar={() => handleVoto(opcion.id)}
/>

Option.jsx — llama la función:

jsx
function Option({ onVotar }) {
  return (
    <button onClick={onVotar}>
      Votar
    </button>
  )
}

Análisis del Código

📄

4.1 App.jsx — Componente Principal

Estado + lógica + renderizado
App.jsx — completo
import { useState } from 'react'
import Option from './components/Option'
import './App.css'

const opcionesIniciales = [
  { id: 1, texto: 'Opción A', votos: 0 },
  { id: 2, texto: 'Opción B', votos: 0 },
  { id: 3, texto: 'Opción C', votos: 0 },
]

function App() {
  const [opciones, setOpciones] = useState(opcionesIniciales)

  // Incrementa los votos de la opción con ese id
  const handleVoto = (id) => {
    setOpciones(opciones.map(op =>
      op.id === id ? { ...op, votos: op.votos + 1 } : op
    ))
  }

  const totalVotos = opciones.reduce((sum, op) => sum + op.votos, 0)

  const opcionGanadora = opciones.reduce((max, op) =>
    op.votos > max.votos ? op : max, opciones[0])

  return (
    <div className="container">
      <h1>Encuesta</h1>
      <div className="stats">
        <p>Total: <strong>{totalVotos}</strong></p>
        <p>Ganador: <strong>{opcionGanadora.texto}</strong></p>
      </div>
      <div className="options">
        {opciones.map(opcion => (
          <Option
            key={opcion.id}
            texto={opcion.texto}
            votos={opcion.votos}
            onVotar={() => handleVoto(opcion.id)}
          />
        ))}
      </div>
    </div>
  )
}

export default App

handleVoto detallado — dos versiones equivalentes:

Versión larga (más clara)
Versión compacta (ternario)
jsx
const handleVoto = (id) => {
  setOpciones(
    opciones.map(op => {
      if (op.id === id) {
        return { ...op, votos: op.votos + 1 } // spread + nuevo votos
      } else {
        return op // sin cambios
      }
    })
  )
}
jsx
const handleVoto = (id) => {
  setOpciones(
    opciones.map(op =>
      op.id === id
        ? { ...op, votos: op.votos + 1 }
        : op
    )
  )
}

reduce() — total de votos explicado visualmente:

Con for (más código)
Con reduce (elegante)
js
let total = 0
for (let i = 0; i < opciones.length; i++) {
  total += opciones[i].votos
}
js
const totalVotos = opciones.reduce((acum, op) => {
  return acum + op.votos
}, 0)   // ← 0 es el valor inicial del acumulador

reduce() para el ganador — cómo itera:

Iteraciónmax actualop.votos¿Mayor?Resultado
1 (inicial){votos: 0}5—{id:1, votos:5}
2{id:1, votos:5}3No{id:1, votos:5}
3{id:1, votos:5}8Sí ✓{id:3, votos:8}

05

Flujo de Datos

Cómo viajan los datos y los eventos en la aplicación — de arriba a abajo y de vuelta arriba.

1
Usuario hace clic en "Votar" El botón en Option.jsx detecta el evento click
2
onClick={onVotar} se activa Option.jsx llama a la función que recibió como prop
3
handleVoto(id) se ejecuta en App.jsx La función del padre recibe el id de la opción votada
4
setOpciones(map(...)) actualiza el estado Se crea un nuevo array con los votos incrementados (inmutabilidad)
5
React re-renderiza la interfaz Todos los Option reciben las nuevas props. Los totales se recalculan. La pantalla se actualiza.
Datos: siempre hacia abajo — El padre pasa props al hijo. El hijo nunca modifica el estado del padre directamente.
Eventos: siempre hacia arriba — El hijo llama al callback del padre. El padre decide qué hacer con esa notificación.

Añade un botón "Resetear" que ponga todos los votos a 0.

💡 Usa setOpciones(opcionesIniciales) o un map() que ponga todos los votos a 0

03

Mostrar porcentaje de votos

📄 Option.jsx — añadir cálculo

Pasa el totalVotos como prop a Option y muestra el porcentaje de votos de cada opción.

💡 Fórmula: Math.round((votos / totalVotos) * 100). Recuerda manejar el caso totalVotos === 0
App.jsx — pasar totalVotos
<Option
  key={opcion.id}
  texto={opcion.texto}
  votos={opcion.votos}
  totalVotos={totalVotos}   // ← nueva prop
  onVotar={() => handleVoto(opcion.id)}
/>
Option.jsx — mostrar porcentaje
function Option({ texto, votos, totalVotos, onVotar }) {
  const porcentaje = totalVotos > 0
    ? Math.round((votos / totalVotos) * 100)
    : 0
  return (
    <div className="option-card">
      <span>{texto}</span>
      <span>{votos} votos ({porcentaje}%)</span>
      <button onClick={onVotar}>Votar</button>
    </div>
  )
}
04

Ordenar por votos

📄 App.jsx — modificar el renderizado

Que las opciones aparezcan siempre de mayor a menor número de votos.

💡 Añade .sort((a, b) => b.votos - a.votos) antes del .map() — pero sobre una copia del array ([...opciones])
App.jsx — Solución
// En el JSX, cambia opciones.map(...) por:
{[...opciones]
  .sort((a, b) => b.votos - a.votos)  // mayor a menor
  .map(opcion => (
    <Option key={opcion.id} ... />
  ))}

// IMPORTANTE: [...opciones] crea una copia.
// sort() muta el array original, lo que violaría la inmutabilidad.
05

Mostrar "Sin votos" cuando no hay votos

📄 App.jsx — renderizado condicional

Si nadie ha votado todavía, muestra "Sin votos todavía" en lugar del nombre del ganador.

💡 Usa un ternario: totalVotos === 0 ? "Sin votos todavía" : opcionGanadora.texto
App.jsx — Solución
// Cambia la línea del ganador en el JSX:
<p className="winner">
  Ganador actual: 
  <strong>
    {totalVotos === 0
      ? 'Sin votos todavía'
      : opcionGanadora.texto
    }
  </strong>
</p>
