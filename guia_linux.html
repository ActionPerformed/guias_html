01 Navegación y estructura de directorios 20 min ▶

Linux tiene un árbol de directorios que parte de / (raíz). No hay letras de unidad como en Windows.

Analogías Windows ↔ Linux

Windows
C:\
→
Linux
/
Windows
C:\Users
→
Linux
/home
Windows
dir
→
Linux
ls

Comandos básicos

bash
pwd # ¿Dónde estoy? (como cd sin args en Windows)
ls -la # listado detallado con permisos y tamaño
cd / # ir a la raíz
cd /home # ir a la carpeta de usuarios
cd ~ # atajo: tu carpeta personal
cd .. # subir un nivel

Estructura del sistema de archivos

/ → raíz (como C:\ pero sin letra de unidad)
├── home/ → carpetas de usuarios (como C:\Users)
│ └── alumno/ → tu espacio personal
├── etc/ → configuraciones del sistema (.ini + registro)
├── var/
│ └── log/ → logs del sistema (como Visor de Eventos)
├── bin/ → comandos básicos (ls, cp, mv...)
└── tmp/ → temporales (se borran al reiniciar)
⚡ EJERCICIO RÁPIDO: Navega a /etc y ejecuta ls | grep "passwd" para ver los archivos relacionados con usuarios.
02 Gestión de archivos 20 min ▶
bash
touch prueba.txt # crear archivo vacío
mkdir proyecto # crear carpeta
cp prueba.txt proyecto/ # copiar archivo
mv prueba.txt nuevo.txt # mover o renombrar
rm prueba.txt # eliminar archivo (¡sin papelera!)
rm -r proyecto/ # eliminar carpeta con todo su contenido
⚠️ PELIGRO: rm -rf es irreversible. No hay papelera de reciclaje. NUNCA ejecutes rm -rf / ni rm -rf /* — borrarías todo el sistema sin confirmación.
💡 EJERCICIO: Crea la estructura mis_notas/ con los archivos tema1.txt y tema2.txt. Luego copia todo a una carpeta backup/.

Ejemplos clave

bash
chmod 755 script.sh # rwxr-xr-x → típico para scripts
chmod 644 readme.txt # rw-r--r-- → típico para archivos
chmod 600 secreto.txt # rw------- → solo tú
chmod u+x script.sh # añade ejecución solo al propietario
REGLA MNEMOTÉCNICA
4
r · leer
2
w · escribir
1
x · ejecutar
7
rwx (4+2+1)

04 Instalación de aplicaciones (apt) 15 min ▶
bash
sudo apt update # actualiza lista de paquetes (no instala nada)
sudo apt upgrade # actualiza todo el sistema
sudo apt install neofetch # instalar una aplicación
neofetch # ejecutarla
sudo apt remove neofetch # desinstalar
WINDOWS
Descargas .exe de webs aleatorias → riesgo de virus, sin control de versiones, desinstalación parcial
LINUX (APT)
Repositorios oficiales → software verificado, dependencias automáticas, desinstalación limpia

Errores típicos y soluciones

Permission denied
No tienes permisos. Usa sudo o revisa con ls -l y ajusta con chmod.
command not found
El comando no existe o hay un error tipográfico. Si debería existir, instálalo con apt install.
rm: cannot remove
Sin permiso de escritura en el archivo, o el archivo no existe. Verifica la ruta con ls.
No such file or directory
La ruta es incorrecta. Usa pwd para saber dónde estás y ls para ver qué hay.

Ejercicio integrador: Analizador de Logs

Ejercicio final · 20 min
Analizador de Logs del Sistema
1
Crear la estructura de carpetas del proyecto
bash
mkdir proyecto_logs
cd proyecto_logs
mkdir datos resultados
2
Crear un archivo de log simulado con errores e información
bash
echo "ERROR: conexión fallida" > datos/sistema.log
echo "INFO: usuario login" >> datos/sistema.log
echo "ERROR: disco lleno" >> datos/sistema.log
3
Copiar y filtrar solo los errores con grep
bash
cp datos/sistema.log resultados/
cd resultados
grep "ERROR" sistema.log > errores.txt
4
Asignar permisos correctos a archivos y carpetas
bash
chmod 644 errores.txt # dueño: rw, resto: r
chmod 755 ../datos # carpeta con permisos de ejecución (acceso)
5
Instalar la herramienta figlet para generar texto decorativo
bash
sudo apt update
sudo apt install figlet -y
6
Generar el reporte final combinando cabecera + errores
bash
figlet "MIS ERRORES" > reporte.txt
cat errores.txt >> reporte.txt
cat reporte.txt
