§01

¿Qué es Lean?

Lean es un software de código abierto que permite a matemáticos y programadores formalizar razonamientos. Nació en Microsoft Research y hoy cuenta con una comunidad activa detrás de Mathlib, una vasta biblioteca de matemáticas formalizadas.

λ
Lenguaje de programación
Funcional de propósito general, similar a Haskell o ML, con un sistema de tipos mucho más potente.
⊢
Asistente de pruebas
Escribes teoremas y demostraciones que el programa verifica automáticamente. Si hay un error, lo señala.
⚙
Kernel de confianza
El núcleo es pequeño y simple — eso garantiza que la verificación de pruebas sea absolutamente fiable.
Idea central: en Lean, una demostración matemática no es un argumento persuasivo escrito en prosa — es un programa que el ordenador ejecuta y verifica byte a byte. No hay ambigüedad posible.

§03

Fundamentos: Tipos y Proposiciones

En Lean, todo tiene un tipo — tanto los datos numéricos como las proposiciones matemáticas.

3.1 — Tipos básicos

lean
-- Definimos una variable de tipo Nat
def mi_numero : Nat := 42

-- Definimos una función que suma dos Naturales
def suma (a b : Nat) : Nat := a + b

-- Lean puede inferir el tipo automáticamente
def otro_numero := 100

3.2 — Explorador de tipos — haz clic para ver el resultado de #check:

#check — comprueba el tipo de cualquier término
Nat
2 + 2 = 4
True
42
suma
Prop
#check Nat ⟶ Nat : Type
Observación clave: Nat tiene tipo Type, pero 2 + 2 = 4 tiene tipo Prop. Las proposiciones matemáticas son, ellas mismas, tipos — esto es la semilla de todo lo que viene en la sección 04.
§04

Proposiciones como Tipos

La idea central de Lean —basada en la Teoría de Tipos Dependientes— es la Correspondencia de Curry-Howard: las proposiciones matemáticas se tratan como tipos, y sus demostraciones son programas que construyen elementos de esos tipos.

Lógica / Matemáticas
Una proposición
Una demostración
"P implica Q"
"P y Q"
Lean / Teoría de Tipos
Un tipo
Un término que "habita" ese tipo
Una función P → Q
Un par (p : P, q : Q)
lean
#check Nat        -- Nat : Type
#check 2 + 2 = 4  -- 2 + 2 = 4 : Prop
#check True       -- True : Prop
En una frase: demostrar un teorema en Lean es literalmente escribir un programa cuyo tipo es el enunciado del teorema. Si el programa compila, la prueba es correcta.
§05

Comenzando a Probar: Tácticas

En vez de escribir el programa de demostración directamente, Lean ofrece tácticas (by ...): comandos que indican cómo construir la prueba paso a paso. Al escribir by, Lean entra en "modo de construcción" — el objetivo es "rellenar el hueco" de la prueba.

TácticaQué haceCuándo se usa
rflDemuestra que algo es igual a sí mismo (reflexividad)El objetivo es literalmente a = a
introIntroduce una hipótesis para probar una implicación P → QEl objetivo tiene forma P → Q
exactAplica un término/hipótesis que coincide exactamente con el objetivoYa tienes la prueba exacta a mano
rwReescribe una parte de la expresión usando una igualdadQuieres sustituir una subexpresión
simpAplica automáticamente reglas de simplificación comunesLa expresión se puede simplificar con lemas conocidos

5.1 — rfl (Reflexividad)

lean
-- Teorema: para todo número natural n, n es igual a n
example (n : Nat) : n = n := by
  rfl  -- "reflectividad": algo es igual a sí mismo

5.2 — intro (Introducción de hipótesis)

lean
example (n : Nat) : n = n → 1 = 1 := by
  intro h       -- introducimos la hipótesis h : n = n
  rfl           -- "1 = 1" se demuestra por reflexividad

5.3 — exact (Aplicación de hechos)

lean
example (n : Nat) (h : n = 5) : n = 5 := by
  exact h

5.4 — rw y simp (Descomposición)

lean — rw (reescribir)
example (n : Nat) : n + 0 = n := by
  rw [Nat.add_zero n]  -- reescribe n + 0 como n
  rfl                  -- ahora tenemos n = n
lean — simp (simplificar)
example (n m : Nat) : n + 0 + m = m + n := by
  simp [Nat.add_zero, Nat.add_comm]
  -- Nat.add_comm es la regla conmutativa: n + m = m + n
  -- Lean reordena y simplifica automáticamente
§06

Simulador de Estado de Prueba

Así es como se ve trabajar con Lean en VS Code: cada táctica transforma el estado de la prueba (las hipótesis arriba del turnstile ⊢, el objetivo debajo). Pulsa cada línea o usa los controles para avanzar.

Tácticas (haz clic para saltar)

1  intro h
2  rfl
Estado actual

Lo que acabas de ver es exactamente el flujo de trabajo en Lean: cada táctica reduce el objetivo hasta que coincide con algo ya demostrado o es trivialmente cierto (rfl). Cuando no queda objetivo, la prueba está completa.
§07

Librería Matemática: Mathlib

Mathlib es una biblioteca masiva y colaborativa de matemáticas formalizadas: álgebra, análisis, topología, probabilidad y mucho más. Es la razón principal por la que Lean es tan potente — la mayoría de los teoremas que necesitas ya están demostrados.

lean — usando un teorema de Mathlib
import Mathlib.Data.Real.Basic  -- para trabajar con números reales

-- Ejemplo: la suma de dos reales positivos es positiva
example (x y : ℝ) (hx : x > 0) (hy : y > 0) : x + y > 0 := by
  exact add_pos hx hy  -- add_pos ya está demostrado en Mathlib
add_pos no es magia — es un teorema que alguien de la comunidad ya demostró y aportó a Mathlib. Tu trabajo, muchas veces, es encontrar el lema correcto, no reinventarlo.
§08

Ejercicios Propuestos

Intenta deducir la secuencia de tácticas antes de revelar la solución. Razona el estado de la prueba en cada paso como hiciste en el simulador.

01

Reflexividad simple

rfl
example (x : Nat) : x + 1 = x + 1 := by ___

¿Qué táctica única demuestra este objetivo?

lean — solución
example (x : Nat) : x + 1 = x + 1 := by
  rfl  -- ambos lados son sintácticamente idénticos
02

Implicación con intro

intro · rfl
example (p : Prop) : p → p := by ___

El objetivo es una implicación. Necesitas introducir la hipótesis y luego usarla directamente.

lean — solución
example (p : Prop) : p → p := by
  intro hp   -- hp : p
  exact hp  -- el objetivo ahora es exactamente "p", que es hp
03

Usar una hipótesis exacta

exact
example (n : Nat) (h : n = 10) : n = 10 := by ___

Ya tienes la hipótesis exacta. ¿Qué táctica la aplica directamente?

lean — solución
example (n : Nat) (h : n = 10) : n = 10 := by
  exact h
04

Reescritura con rw

rw · rfl
example (a b : Nat) (h : a = b) : a + 1 = b + 1 := by ___

Usa la hipótesis h para reescribir a como b en el objetivo.

lean — solución
example (a b : Nat) (h : a = b) : a + 1 = b + 1 := by
  rw [h]  -- sustituye a por b: objetivo pasa a "b + 1 = b + 1"
  -- rw cierra automáticamente con rfl si el resultado es trivial
05

Cadena completa de tácticas

intro · rw · rfl
example (n : Nat) : n = 0 → n + 5 = 5 := by ___

Combina varias tácticas: primero introduce la hipótesis, luego reescribe usándola.

lean — solución
example (n : Nat) : n = 0 → n + 5 = 5 := by
  intro h     -- h : n = 0
  rw [h]    -- objetivo: 0 + 5 = 5
  -- rw cierra automáticamente porque 0 + 5 se reduce a 5
